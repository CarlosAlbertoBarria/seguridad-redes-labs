# Checklist de análisis de topología

## 1. Relevamiento

- ¿Qué dispositivos existen?
- ¿Qué función cumple cada dispositivo?
- ¿Qué enlaces conectan los dispositivos?
- ¿Qué servicios dependen de cada equipo?
- ¿Qué usuarios o áreas utilizan la red?

## 2. Activos a proteger

- Servidores.
- Equipos de usuarios.
- Datos críticos.
- Acceso a Internet.
- Servicios internos.
- Equipos de red.
- Credenciales.
- Disponibilidad del servicio.

## 3. Puntos de incidencia

- ¿Existe un punto único de falla?
- ¿Qué ocurre si cae el switch principal?
- ¿Qué ocurre si cae el router?
- ¿Qué ocurre si falla el firewall?
- ¿Qué ocurre si se corta el enlace a Internet?
- ¿Hay redundancia?
- ¿Hay respaldo eléctrico?

## 4. Seguridad

- ¿Existe segmentación de red?
- ¿Hay firewall?
- ¿Hay IDS o IPS?
- ¿Hay control de accesos?
- ¿Hay reglas ACL?
- ¿Hay monitoreo?
- ¿Hay backups?
- ¿Hay registros o logs?

## 5. Mejoras posibles

- VLANs.
- ACLs.
- Firewall perimetral.
- DMZ.
- Redundancia.
- UPS.
- Backup.
- Monitoreo.
- Segmentación.
- Alta disponibilidad.

## 6. Conclusión

Indicar si la topología cumple criterios básicos de seguridad, disponibilidad y administración.
